# Contributing

Frontier Max accepts changes that make benchmark evidence or routing policy more legible, reproducible, and privacy-preserving.

Before opening a pull request:

1. Explain the workload policy, evidence gap, or failure mode the change addresses.
2. Link primary documentation or source evidence behind any benchmark or routing claim.
3. Preserve raw source values and expose every transformation.
4. Keep prompts, repository content, diffs, API keys, and model responses out of telemetry and fixtures.
5. Run `npm run lint` and `npm test` from the repository root.

Provider sponsorship, affiliate economics, or commercial relationships must never alter benchmark interpretation, route eligibility, or route ordering. Disclose any relevant conflict in the pull request.
