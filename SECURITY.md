# Security policy

Do not include prompts, repository content, diffs, model responses, API keys, authentication tokens, or private benchmark data in a public report.

Report suspected vulnerabilities through the repository's private security-advisory flow once the public repository is connected. Include the affected version, a minimal content-free reproduction, and expected impact. Do not open a public issue for credential exposure, command execution, path traversal, symlink, or receipt-integrity problems before maintainers have had a reasonable opportunity to respond.
