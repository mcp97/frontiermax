### Description

Benchmarks tell us who won one evaluation contract. Frontier Max helps decide what should run the actual work.

In this one-minute demo, Frontier Max:

- preserves source-linked benchmark evidence;
- uses Gemini 3.5 Flash to interpret a short coding task;
- validates the result against a versioned workload policy;
- requests an interactive or delegated OpenRouter Pareto Code route for OpenCode; and
- writes a content-free local decision receipt.

For interactive coding, the current policy requests the fastest available model in OpenRouter’s high Artificial Analysis coding tier, ranked by p50 throughput. For delegated coding, it requests the cheapest available model in that tier. Frontier Max does not proxy prompts, code, keys, diffs, or model responses, and it does not claim to observe the concrete underlying model selected by OpenRouter.

## LinkedIn

We have spent years making AI benchmarks more sophisticated, but almost no time making them operational.

A benchmark can tell you which system won under one evaluation contract. It cannot, by itself, answer a production question: which model policy should run this task under our quality bar, budget, waiting-time tolerance, context requirement, and risk constraints?

That is the problem behind **Frontier Max**.

Frontier Max preserves source-linked benchmark evidence, makes the contract behind a score legible, and turns a real task into an inspectable workload policy. In our first coding workflow, Gemini 3.5 Flash interprets whether the work is interactive or delegated. Deterministic code validates that choice, then requests the corresponding OpenRouter Pareto Code route for OpenCode and creates a content-free local receipt.

The distinction matters:

- When a human is pairing, waiting time matters.
- When a job can run overnight, model price can matter more after quality is gated.
- Tokens remain measured, but they should be optimized only when context, transfer time, or token cost is actually binding.

Gemini interprets semantics; it does not invent an executable route. Frontier Max does not proxy prompts, repositories, keys, diffs, or model responses.

We built this for the Stanford × DeepMind Gemini Hackathon. I would love feedback from AI platform, engineering productivity, evaluation, and FinOps teams.
