# Security policy

Do not include prompts, repository content, diffs, model responses, API keys, or authentication material in a public report.

For a suspected vulnerability, use the repository's private security-advisory flow once available. Include the affected version, a minimal content-free reproduction, and the expected impact. Please do not open a public issue for credential exposure, command execution, path traversal, symlink, or receipt-integrity problems before maintainers have had a reasonable opportunity to respond.
