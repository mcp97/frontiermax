# Contributing

Frontier Max accepts changes that make routing policy more legible, reproducible, or privacy-preserving.

Before opening a pull request:

1. Explain which workload policy or failure mode the change addresses.
2. Link the primary documentation or benchmark evidence behind any routing claim.
3. Keep prompts, source code, diffs, API keys, and model responses out of telemetry and fixtures.
4. Run `npm test` from this directory.

Model-provider sponsorship, affiliate economics, or commercial relationships must never alter route eligibility or ordering. Disclose any relevant conflict in the pull request.
